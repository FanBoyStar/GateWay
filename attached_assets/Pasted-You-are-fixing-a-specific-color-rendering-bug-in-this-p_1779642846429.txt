## THE BUG
The primary pink color (#E8186D) renders correctly as pink on desktop browsers 
but shifts to red on mobile browsers (iOS Safari and Chrome Android). This is 
caused by the color sitting on the sRGB/P3 gamut boundary, which causes mobile 
wide-gamut displays to misinterpret it.

## THE FIX — Apply ALL three of these changes

### CHANGE 1 — index.html
Locate the <meta name="viewport"> tag in index.html and make sure it reads 
exactly this — do not remove any existing attributes, only add viewport-fit=cover 
if it is missing:

<meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover">

Also add this new meta tag directly below it if it does not already exist:

<meta name="color-scheme" content="light dark">

### CHANGE 2 — index.css
Locate every instance of the color #E8186D in index.css (it will appear under 
--color-primary in the :root block). Replace the entire --color-primary line 
with these two lines — keep both, the first is the fallback, the second is the 
explicit sRGB fix:

--color-primary: #E5177A;
--color-primary: color(srgb 0.898 0.090 0.478);

Do the same inside the html.light block where --color-primary is set to #C8005A. 
Replace that line with:

--color-primary: #C4006B;
--color-primary: color(srgb 0.769 0.000 0.420);

### CHANGE 3 — index.css
Add this block at the very bottom of index.css, after all existing rules. This 
targets mobile viewports specifically as a safety net in case the color() function 
is not supported:

@media (max-width: 767px) {
  :root {
    --color-primary: #E5177A;
    --color-primary-soft: rgba(229, 23, 122, 0.15);
    --color-primary-glow: rgba(229, 23, 122, 0.35);

  html.light {
    --color-primary: #C4006B;
    --color-primary-soft: rgba(196, 0, 107, 0.10);
    --color-primary-glow: rgba(196, 0, 107, 0.20);
  }
}
